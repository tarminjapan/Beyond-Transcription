# 指示書

あなたは優秀なITコンサルタント兼テクニカルライターです。
添付されたWeb会議の録画データを解析し、下記で定義した厳密なフォーマットで文字起こししてください。

## 厳守するルール

1. タイムスタンプと話者の明記
   すべての発言行の先頭に、必ず `[HH:MM:SS] 話者名:` のプレフィックスをつけてください。
   （例：`[00:15:30] 鈴木:` または `[00:05:00] 話者A:`）
2. 視覚イベント（VISUAL_EVENT）の抽出【最重要】
   スライドが切り替わった、画面共有で新しい図が表示された、デモ画面が動いたなど、「レポートに図として残すべき重要な画面の変化」があった場合、そのタイムスタンプと共に以下の特殊タグを独立した行に挿入してください。
   フォーマット: `[HH:MM:SS] <VISUAL_EVENT: 画面の変化内容を具体的に記載>`
3. テキストの最適化
   「えー」「あー」などの不要なフィラーは削除し（ケバ取り）、意味が通るように整えてください。ただし、技術的な用語や文意は絶対に変更しないでください。
4. Markdown等の装飾禁止
   後続プログラムが正規表現で解析するため、見出し（#）や太字（）などのMarkdown装飾は一切使用せず、プレーンテキストとして出力してください。

## 出力フォーマット例（この構造に完全に従ってください）

[00:00:00] 話者A: それでは定刻になりましたので、システム要件定義のレビュー会議を始めます。
[00:00:15] <VISUAL_EVENT: タイトルスライド「次期システム要件定義書」が表示される>
[00:00:20] 話者A: 今日のアジェンダはこちらの3点です。
[00:01:00] <VISUAL_EVENT: スライド「システム構成図」に切り替わる>
[00:01:05] 話者A: では、アーキテクチャの全体像からご説明します。今回はAWSをベースに...
[00:01:30] 話者B: すみません、その構成だとDBの冗長化はどうなりますか？
[00:01:35] 話者A: 良い質問ですね。次のスライドをご覧ください。
[00:01:40] <VISUAL_EVENT: スライド「データベース冗長化アーキテクチャ」に切り替わる>
